Classes and Ids

When and How to use Classes and Ids

Lets talk about classes and ids, what they do and how you can use them when building and creating a webpage.

Classes and Ids can be used to help style individual elements and give them different properties. Classes are the more common of the two for styling but Ids have a unique use as well. Let’s break them both down!

Ids

The first thing to know about Ids is that they are unique. The id attribute is used to identify a specific element. Ids can used to apply styling through CSS similar to classes and are commonly used in HTML to help navigate a user around a webpage. They are particularly helpful for a user to jump to a certain section so they can avoid scrolling to the area they want.

Here's an Example:

You have a webpage with a step-by-step guide to making a lasagne. A user comes to your page just to find out how to make cheese sauce. Time is of the essence! They don’t want to scroll through the entire page to find the cheese sauce step, so they see a list with each step in a contents section. By clicking on ‘Step 6 - Cheese Sauce’ this takes them directly to that part of your webpage, making their user experience that much easier.

Classes

Classes are an attribute used to define elements and helps create unique styling and properties. Unlike ids, classes can be used across many elements and can are extremely useful when it comes to designing your webpage and implementing consistency among elements. Imagine you want to have a group of headings to be bright red, your images to be the same size, and your buttons to have the same font. You can create classes with those attributes and apply it to the elements you want to style. You can create and add classes to just about any element you can think of!

Summary

You can see how powerful both these tools can be and why they are widely for web development. The best thing of all is can you choose how to implement these into your webpage. You can have an id to simply help a user navigate around your page, or add a one-off unique style. You can create a class with basic style properties or something so complex that nobody has ever done before!
